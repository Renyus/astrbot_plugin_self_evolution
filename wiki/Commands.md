## 系统命令

### /system help

显示插件帮助信息。

**使用场景**：
- 刚安装插件，确认是否正常工作
- 忘记命令格式时查看

**示例**：
```
/system help
```

**输出示例**：
```
自我进化插件 v1.x.x

可用命令：
/system help - 显示此帮助
/system version - 显示版本
/reflect - 手动触发反思
...
```

---

### /system version

显示插件版本信息。

**使用场景**：
- 排查问题时确认版本
- 检查是否是最新版

**示例**：
```
/system version
```

## 用户画像命令

### /profile view [用户ID]

查看用户画像。

**使用场景**：
- 想知道机器人"记住"了你什么
- 检查画像是否生效

**示例**：
```
# 查看自己的画像
/profile view

# 查看指定用户的画像（需要权限）
/profile view 123456789
```

**💡 什么时候用这个命令？**
- 刚聊了很久，想看看是否形成了画像
- 觉得机器人对你"不够了解"

---

### /profile create [用户ID]

手动创建用户画像。

**使用场景**：
- 自动建档未生效时手动创建
- 想立即为某个用户建立档案

**示例**：
```
# 为自己创建画像
/profile create

# 为指定用户创建画像（需要权限）
/profile create 123456789
```

**⚠️ 注意**：创建过程需要分析历史消息，可能需要几秒到几十秒。

---

### /profile update [用户ID]

手动更新用户画像。

**使用场景**：
- 画像信息过时了，想立即更新
- 刚聊了很多关于自己的事

**示例**：
```
/profile update
```

---

### /profile delete <用户ID>

删除用户画像（管理员）。

**使用场景**：
- 画像信息有误，想重置
- 用户要求删除自己的数据

**示例**：
```
/profile delete 123456789
```

---

### /profile stats

查看画像统计（管理员）。

**使用场景**：
- 了解有多少用户已建档
- 排查画像功能是否正常

**示例**：
```
/profile stats
```

## SAN精力系统命令

### /san show

查看当前 SAN 状态。

**使用场景**：
- 机器人表现"疲惫"时检查
- 想知道精力恢复了多少

**示例**：
```
/san show
```

**输出示例**：
```
当前SAN值：45/100
状态：正常
恢复速率：10/小时
```

**💡 什么时候用这个命令？**
- 机器人回复变短或拒绝服务时
- 想了解机器人的"精力"状况

---

### /san set [值]

设置当前 SAN 值（管理员）。

**使用场景**：
- 测试 SAN 低值时的行为
- 手动"恢复"机器人精力

**示例**：
```
# 恢复到满精力
/san set 100

# 设置为低精力状态（测试用）
/san set 10
```

## 情感积分命令

### /affinity show

查看机器人对你的好感度。

**使用场景**：
- 想知道和机器人的"关系"如何
- 好奇之前的互动是否有影响

**示例**：
```
/affinity show
```

**输出示例**：
```
当前好感度：15
等级：友好
下次恢复：8小时后
```

---

### /affinity debug <用户ID>

查看详细好感度状态（管理员）。

**使用场景**：
- 排查好感度计算问题
- 查看触发记录和冷却状态

**示例**：
```
/affinity debug 123456789
```

---

### /set_affinity <用户ID> <分数>

手动设置好感度（管理员）。

**使用场景**：
- 测试不同好感度下的回复差异
- 手动修正异常的好感度

**示例**：
```
# 将某用户好感度设为100
/set_affinity 123456789 100

# 重置为0
/set_affinity 123456789 0
```

## 反思命令

### /reflect

手动触发自我反思。

**使用场景**：
- 想立即总结刚才的对话
- 测试反思功能是否正常

**示例**：
```
/reflect
```

**💡 什么时候用这个命令？**
- 聊了很久重要的事，想立即记录
- 感觉机器人的回应不够"贴切"

**⚠️ 注意**：反思需要分析历史消息，可能需要一些时间。

## 表情包命令（管理员）

### /sticker list [页码]

查看表情包列表。

**使用场景**：
- 查看有哪些可用表情包
- 获取表情包的 UUID

**示例**：
```
/sticker list
/sticker list 2
```

---

### /sticker add

添加表情包。

**使用场景**：
- 想添加新的表情包到库中

**使用步骤**：
```
1. 发送图片
2. 立即发送：/sticker add
3. 系统会提示添加成功
```

---

### /sticker delete <UUID>

删除指定表情包。

**示例**：
```
/sticker delete abc-123-def
```

---

### /sticker preview <UUID>

预览指定表情包。

**示例**：
```
/sticker preview abc-123-def
```

---

### /sticker stats

查看表情包统计。

**使用场景**：
- 了解库存数量
- 查看发送冷却状态

**示例**：
```
/sticker stats
```

---

### /sticker sync

同步本地表情包目录。

**使用场景**：
- 手动添加了图片到目录后同步
- 修复表情包显示问题

**示例**：
```
/sticker sync
```

---

### /sticker clear

清空所有表情包。

**⚠️ 警告**：此操作不可恢复！

**示例**：
```
/sticker clear
```

## 数据库命令（管理员）

### /db status

查看数据库状态。

**示例**：
```
/db status
```

---

### /db reset

重置数据库。

**⚠️ 警告**：会清空所有数据，但会保留知识库！

**使用场景**：
- 数据损坏需要重建
- 想完全重新开始

**示例**：
```
/db reset
```

---

### /db rebuild

重建数据库结构。

**使用场景**：
- 升级后数据结构变化
- 出现数据库错误时修复

**示例**：
```
/db rebuild
```

## 其他命令

### /shut [分钟]

让 AI 在当前群暂停响应。

**使用场景**：
- 临时不想让机器人参与
- 群里需要安静时

**示例**：
```
# 查看当前状态
/shut

# 暂停30分钟
/shut 30

# 取消暂停
/shut 0
```

**示例**：
```
/今日老婆
```
