# 记忆系统

> **一句话理解**：让机器人像认识老朋友一样记住每个人的喜好和群里的重要约定。

---

## 记忆系统能做什么？

### 场景 1：记住用户喜好

```
用户A：我喜欢吃火锅
...（几天后）...
用户A：今晚吃啥？
机器人：你之前说过喜欢吃火锅，要不要考虑？
```

### 场景 2：记住群里的约定

```
群里：我们周末一起去爬山吧，周日上午9点集合
...（周末）...
用户B：我们什么时候说来着？
机器人：群里之前约定周日上午9点集合去爬山
```

### 场景 3：每日总结

```
用户：昨天群里聊了什么？
机器人：昨天主要讨论了...
```

---

## 记忆类型

### 1. 人物画像（Profile）

**记住什么**：这个人是谁、喜欢什么、什么性格

**自动记录的内容**：
- 身份信息（学生/上班族等）
- 兴趣爱好（游戏、食物等）
- 行为特征（话多/话少等）

**示例**：
```
用户：我是个程序员，平时喜欢打原神
→ 自动记录到画像：职业=程序员，爱好=原神
```

**查看方式**：
```
/profile view
```

---

### 2. 会话事件（Session Event）

**记住什么**：群里的约定、决定、安排

**适合记录**：
- ✅ "我们下周日聚会"
- ✅ "决定用Python写这个项目"
- ✅ "群规：禁止发广告"
- ❌ "我喜欢吃辣"（这是画像，不是事件）

**查询方式**：
```
用户：我们之前约定什么时候？
→ 机器人调用工具查询事件记忆
```

---

### 3. 每日总结

**记住什么**：某一天这个群/私聊整体聊了什么

**生成时间**：默认每天凌晨3点自动总结前一天

**查询方式**：
```
用户：昨天群里聊了什么？
机器人：昨天大家主要讨论了...
```

## 记忆如何存储

```
┌─────────────────────────────────────┐
│         用户画像                     │
│   存储位置：本地文件                 │
│   路径：data/self_evolution/profiles/│
├─────────────────────────────────────┤
│         会话总结/事件                │
│   存储位置：AstrBot知识库            │
│   按群隔离：g_群号 / p_用户号        │
├─────────────────────────────────────┤
│         反思/积分/SAN               │
│   存储位置：SQLite数据库             │
│   路径：data/self_evolution/*.db    │
└─────────────────────────────────────┘
```

## 使用示例

### 示例 1：建立画像

```
用户：我叫小明，是个大学生，喜欢打篮球
[系统自动分析并更新画像]

几天后...
用户：推荐点运动
机器人：你之前说过喜欢打篮球，可以试试...
```

### 示例 2：记录约定

```
群里：@机器人 记得提醒我们周五开会
用户B：对，周五下午3点
[系统记录为session_event]

周五...
用户：我们什么时候开会？
机器人：群里之前约定周五下午3点开会
```

### 示例 3：查询历史

```
用户：昨天群里聊了什么？
→ 机器人调用 get_group_memory_summary
→ 返昨日总结

用户：他以前说过什么？
→ 机器人调用 get_user_messages
→ 返回该用户历史发言
```

## 记忆写入规则

不是所有对话都会被记录，系统会智能判断：

```
输入内容
    │
    ▼
┌─────────────────────────────────────┐
│ 包含"约定"、"决定"、"时间"等关键词？  │
│ → 是：保存为会话事件                 │
└─────────────────────────────────────┘
    │ 否
    ▼
┌─────────────────────────────────────┐
│ 是身份信息/偏好/性格描述？            │
│ → 是：更新用户画像                   │
└─────────────────────────────────────┘
    │ 否
    ▼
┌─────────────────────────────────────┐
│ 是语气/风格纠偏？                    │
│ → 是：保存为反思提示                 │
└─────────────────────────────────────┘
    │ 否
    ▼
   丢弃
```

## 相关配置

```json
{
  "memory_summary": {
    "memory_enabled": true,              // 启用长期记忆
    "memory_kb_name": "self_evolution_memory",  // 知识库名称
    "memory_summary_schedule": "0 3 * * *",     // 每日总结时间
    "enable_kb_memory_recall": true       // AI回复时召回记忆
  },
  "profile": {
    "enable_profile_injection": true,    // 回复时注入画像
    "auto_profile_enabled": true,        // 自动创建画像
    "profile_msg_count": 500             // 分析消息条数
  }
}
```

## 相关 LLM 工具

机器人可以通过以下工具主动查询记忆：

| 工具 | 用途 | 示例问题 |
|------|------|---------|
| `get_user_profile` | 获取用户画像 | "这个人是什么样" |
| `get_user_messages` | 查询用户历史发言 | "他以前说过什么" |
| `get_group_recent_context` | 获取最近群聊 | "刚刚在聊什么" |
| `get_group_memory_summary` | 获取某天的总结 | "昨天聊了什么" |
| `upsert_cognitive_memory` | 写入新记忆 | 发现重要信息时自动调用 |

## 故障排查

### 问题：记忆不生效

**检查清单**：
- [ ] `memory_enabled` 已开启
- [ ] 知识库已正确创建
- [ ] `enable_kb_memory_recall` 已开启
- [ ] `enable_profile_injection` 已开启

**调试方法**：
```
1. 开启 memory_debug_enabled
2. 发送测试消息
3. 查看日志 [MemoryWrite] 是否有记录
4. 查看日志 [MemoryInject] 是否有注入
```
